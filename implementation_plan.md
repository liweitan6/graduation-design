# 基于对比实验的故障边界分析 (Fault Boundary Analysis) 实施计划

## 1. 设计初衷与目标

响应导师关于“用成功/失败用例做对比实验，找出故障边界”的研究建议。该理论基于差分调试（Delta Debugging）思想：当深度学习算子发生崩溃时，通过检索高度相似但执行成功的测试用例，对比两者输入张量和算子参数的差异（Delta），可以精准锁定引发系统崩溃的临界变量（如矩阵维度的极限倍数），从而提炼出底层 API 真正的安全运行边界。

## 2. 核心架构逻辑实现

本功能将复用我们已建成的“双库向量检索”与“RAG 大模型分析”能力骨干。

### Phase A: 后端分析控制流开发 (`scripts/data_service.py`)

新增 API: `POST /api/analysis/fault-boundary`

1. **同构对齐召回检索阶段**：
   - 入参：`failed_case_uid`（一个已经确认引发崩溃漏洞的目标测试脚本节点）
   - 处理逻辑：调用 Milvus 获取其目标张量特征，并在向量查询中强制追加属性过滤条件 `expr="status == 'success'"`，召回欧式距离最近的高维“双胞胎”成功对照参照对象 (`successful_case_candidate`)。
2. **结构张量解耦比对（Diff）阶段**：
   - 提取出 Failed 目标与 Successful 目标的原始 JSON 参数表结构。
   - 开发独立的 Diff 比较函数，计算出导致两者运行结果分化的关键参数变更阵列（如：同样包含 Conv2d 算子，失败用例改变了 padding 大小或越界改变了 in_channels）。
3. **大模型（LLM）临界推理边界归纳指令阶段**：
   - 组装专门提炼规则的深层分析提示词（Prompt）：同时塞入正确的网络运行环境、错误的对照崩溃实验数据、捕获的日志报错。
   - 请求 DeepSeek-V3 接口推理输出：由它来直接点名总结该漏洞发生临界线的底层控制阈值表达式。

### Phase B: 可视化对比视图呈现 (`frontend/...`)

1. **对比操作台集成**：
   - 在用户查看特定失败案例的记录看板页面或在专有的纠错关联大厅，置入一颗显眼的 “🔬 启动故障临界线对比实验” 交互探针探测按钮。
2. **视图渲染展示**：
   - 前端接收到接口测算回传的数据后：
     - 上半部：通过横向两幅类似 GitHub 代码变动审查高亮的面板（左右分布），左绿右红直观提示向审核专家标记出两个控制测试的输入源在哪个人工张量变量参数设定发生了严重越界偏移。
     - 下半部：渲染一个强调引用警告区域组件，完整出具刊载展示由 AI 前线结合图比对出最终裁定的框架使用故障越界防线红线定义分析归纳结果。
