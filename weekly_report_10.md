# 本周工作汇报

本周工作主要是在上周“分析故障边界”的基础上，利用 Daikon 动态不变量推断技术进行故障边界定界。

在上周的工作，我最初打算直接把失败和成功用例的参数差异和日志信息喂给大模型，依赖大模型的去猜测和归纳故障边界的公式。这样把核心的推导工作完全交给大模型可能不太严谨，大模型存在幻觉问题，如果他捏造一个关系式，系统根本没法验证

所以最后系统实现用严格的算法来推导边界。具体的实现是当系统捕捉到一个崩溃用例时，批量召回前 10 个具有相同算子拓扑结构但执行成功的用例，同时再捞出前 10 个失败用例。

接着，系统会在后台把这两组用例的嵌套参数，全部展平成一维的键值对。然后算法去对这些变量进行单变量、双变量以及多变量的排列组合与交叉比对，测试深度学习中常见的关系约束：
*   **单变量物理阈值与奇偶性**：扫描如 `x > 0`、`x >= -1` 等常量越界，以及对 `stride` 等参数的 `x % 2 == 0` 偶数强校验。
*   **双变量倍数与整除关系**：扫描如 `x <= y` 的基础不等式，穷举系数进行 `x <= c*y + d` 的线性探测，以及深度学习中极度常见的整除校验（如探测 `in_channels` 能否被 `groups` 整除的 `x % y == 0`）。
*   **多变量联合空间约束**：针对 CNN 等算子特有的空间坍缩问题，扫描 `x + 2*y >= z` （例如精确捕获 `input_size + 2*padding >= kernel_size`），以及针对 Tensor 变形操作扫描 `x * y == z` 乘积维度守恒。

算法最终会严格筛选出那些在所有成功样例中成立，但在所有失败样例中不成立的决定性约束。

然后将算法硬算出来的不变量公式，塞给大模型。让大模型只负责把底层的数学关系翻译成的自然语言诊断报告。
