# Agent 研发交接指南 (Project Handover Guide)

> **致下一个接手本项目的 Agent 助手**：
> 这是一个面向学术论文（硕士毕业设计）的高级研究型工程项目。该项目旨在针对深度学习框架（如 PyTorch）的底层 API 和算子组合，进行基于模糊测试（Fuzzing）的安全性验证、故障定界以及边界红线推断。系统融合了经典的软件工程分析理论（如差分调试、Daikon 不变量推断）和前沿的大语言模型（LLM）能力。

## 一、 项目背景与核心价值
本系统的核心学术价值并非单纯的“调包生成数据”，而是在“如何优雅、严谨地解释大模型框架为什么会崩溃”。
我们摒弃了“把报错日志直接丢给大模型让它瞎猜”的偷懒做法，而是建立了一条 **“双空间算法硬推断 + 大模型软解释”** 的双轨排障流水线。通过结构特征向量化（Milvus）和拓扑算子级的数学分析，系统能在物理层面证明故障发生的真正数学红线。

## 二、 技术栈架构
项目采用了前后端分离 + 混合后端的架构：
*   **前端界面 (Frontend)**: Vue 3 + TypeScript + Element Plus。页面包括数据大盘、故障溯源分析、边界对比实验窗等极客风 UI。
*   **数据接入侧 (Java Backend)**: 基于 Spring Boot，主要负责基础的 CRUD 路由转发和兼容性代理。
*   **核心算法侧 (Python Backend)**: 基于 FastAPI (`scripts/data_service.py`)，负责计算密集型的 AI 任务，端口为 `5000`。
*   **双库存储**: 
    *   **MySQL**: 存储测试用例的基础元数据（状态、覆盖率、报错日志）。
    *   **Milvus**: 存储使用 `SentenceTransformer` 对算子拓扑结构提取出的高维特征向量，用于跨用例的“相似性克隆体”检索。
*   **大语言模型 (LLM)**: 接入了 DeepSeek-V3 接口，用于语义总结。

## 三、 已完成的四大核心功能模块

### 1. 覆盖率效能评估 (CEI - Coverage Efficiency Index)
系统能对模糊测试产生的数据集进行清洗。通过计算每个测试用例的 `CEI = C_hit / (T_exec * alpha)`，将执行缓慢且对覆盖率提升无贡献的低优用例直接剪枝，从而保障测试集的小而精。

### 2. DFS 拓扑故障溯源 (Fault Locator)
位于 `scripts/fault_locator.py`。当计算图崩溃时，引擎会通过深度优先搜索逆向遍历算子链（构建邻接矩阵），根据抛出的框架错误类型（如 `CUDA_OOM`, `SHAPE_MISMATCH`），精准高亮出嫌疑算子节点并给出置信度。

### 3. 【核心高亮】双空间 Daikon 故障边界定界引擎
位于 `scripts/boundary_analyzer.py` 和 `data_service.py`。这是本系统最具学术价值的亮点：
*   **对照组召回**：当某个用例崩溃时，系统利用 Milvus 找出 10 个拓扑相同但成功的“健康对照组”，同时找出 10 个失败的“崩溃对照组”。
*   **参数展平与空间隔离**：`DaikonInvariantEngine` 会将这两组嵌套的张量参数（如 `kernel_size`, `padding`）全部展平为一维数值键值对。
*   **绝对数学规律探测**：通过多重循环扫描，测试海量算子规则（如 `x > 0`, `in_channels % groups == 0`, `input_size + 2*padding >= kernel_size`）。**只有当某条公式在 10 个成功样本中全票通过，且在 10 个失败样本中全票否决时**，它才会被提取为导致崩溃的绝对数学红线（真正意义上的分水岭）。

### 4. 算法级防线与 LLM 自然语言解读
系统将 Daikon 引擎算出的“绝对数学公式”作为 Context 下发给 DeepSeek 大模型。大模型的角色从“毫无根据地猜边界”降级为“翻译官”，它负责将底层严苛的数学公式转写成人类易读的自然语言排障报告，展示在前端的差分对比面板中。

## 四、 核心代码目录索引

*   `/frontend/`: 前端 Vue 源码目录。
    *   `/src/views/Correlations.vue`: 故障边界对比和 Daikon 算法警告展示的核心页面。
    *   `/src/api/index.ts`: 所有的后端请求路由都在这里。
*   `/scripts/`: Python 算法核心服务目录。
    *   `data_service.py`: FastAPI 服务主入口，包含了 `analyze_fault_boundary` 路由编排逻辑和 Milvus 检索。
    *   `boundary_analyzer.py`: 存放 Daikon 不变量探测引擎的核心算法。
    *   `fault_locator.py`: 拓扑回溯故障定位引擎。
    *   `run_gcov_experiment.sh`: 在 WSL 下对 PyTorch 底层 C++ 源码进行 LCOV 覆盖率验证的实验脚本（该实验已跑完，证明了系统的精简能力）。

## 五、 后续待接手的方向与建议

未来的 Agent 接手该项目后，可以围绕以下方向发力：

1.  **闭环自动化 (Auto-Loop Test Generation)**:
    目前分析出了边界公式，下一步可以考虑写一个变异器（Mutator），依据大模型输出的“修复建议”或“安全边界公式”，自动生成一组**修复后**的新测试用例送回框架重测，形成真正的 Fuzzing 闭环。
2.  **Daikon 模板池的扩充**:
    目前的数学扫描模板（`boundary_analyzer.py` 内）已经涵盖了单变量阈值、奇偶性、双变量等式整除、三变量空间坍缩。如果你有深度学习底层的优化经验，可以在这里增加更多关于内存分配（Strides）相关的扫描模板。
3.  **论文润色与实验图表导出**:
    代码层面的功能已经非常完备。接下来你需要配合 User（研究生），利用这套工具导出大量的真实崩溃实验截图、CEI 对比曲线、以及 Daikon 引擎捕获出的真实数学公式表格，用于填充其毕业论文的实验验证章节。
