# 毕业设计周进展情况记录（第 10-14 周计划与实施）

**课题名称**：面向深度学习模糊测试的测试用例智能管理与效能分析系统  
*(注：第 10-14 周为主体研发结束后的冲刺与答辩准备阶段)*

---

## 第 10 周进展情况记录

### 本周完成/计划情况：
1. **扩大物理验证规模**：在前期 200 例 A/B 物理测速的基础上，利用 `real_cei_executor.py` 发起更大规模的 1000 例深度物理跑测。进一步验证系统在长时间高负载 Fuzzing 环境下的防彻底死锁能力与 CEI 效能截断的稳定性。
2. **前端渲染性能重构**：针对 ECharts 渲染万级 PCA 散点出现的掉帧与卡顿问题，引入 ECharts GL (WebGL) 底层渲染策略，实现十万级测试用例降维分布的丝滑缩放。
3. **论文核心章节动笔**：基于上周跑出的详实数据与图表，正式撰写毕业论文的《系统架构与核心算法设计》以及《实验论证与效能分析》两章核心内容。

### 存在的问题：
在扩大测试规模后，MySQL 与 Milvus 数据库的双写同步压力增大，偶尔出现极个别短时数据状态不一致的现象。

### 下周计划：
在代码层面上增加消息队列或补偿轮询机制来保证数据最终一致性；开启毕业论文前三章（绪论、相关工作、需求分析）的撰写。

## 第 11 周进展情况记录

### 本周完成/计划情况：
1. **大语言模型输出合规化（预演）**：针对 DeepSeek 自动生成的补盲 PyTorch 代码可能存在的语法或过期 API 风险，在系统的 LLM 自动交互环路中增加了局部的“语法树抽象提取验证（AST Validation）”机制，确保送入后端的生成代码基本合规。
2. **全链路端到端联调测试**：打通 Vue3 交互面板指令 -> Java CRUD 数据交互 -> Python 5000 端口（PCA/CEI/LLM运算） -> 触发 WSL 底层 Gcov C++ 物理流的完整体系。消除各端口间的跨域（CORS）与数据包装隐患。
3. **论文初稿拼装**：完成论文《系统实现与微服务联动机制》、《总结与展望》章节。并将前端 UI 截图、RAG 问答链路图示、V3 版本物理 A/B Test 校准图全量排版插入至论文正文中。

### 存在的问题：
论文的论述部分部分结构过于口语化或偏向工程实战，缺乏学术化的包装与严密的理论推演公式陈述。

### 下周计划：
提交给指导老师进行第一轮初版审阅；针对系统整体的代码仓库进行脱敏、瘦身及开源准备。

## 第 12 周进展情况记录

### 本周完成/计划情况：
1. **专家审阅与修改**：根据导师第一轮修改意见，大幅精练了“覆盖率效能指数（CEI）”的数学推论过程。加强了引用文献的数量与权威度，补充了近三年来（2023-2025年）对深度学习 Fuzzing 体系的前沿综述说明。
2. **论文外文翻译工作**：严格校对英文摘要，以及系统架构相关的外文文献引注格式，完成学校规定字数的学术文献翻译任务。
3. **代码冻结与文档沉淀**：完成 `graduation-design` 仓库的代码 Freeze 环节。补充完整的 `README.md`，编写详细的系统安装环境说明、MySQL 表结构 SQL 以及 Milvus 启动依赖组件，并导出项目白皮书供答辩评委审阅。

### 存在的问题：
论文的部分理工类图标要求达到 300DPI 分辨率，并且要求黑白打印清晰，目前通过 Python 绘制的少部分彩色热力图在灰度打印下区分度不高。

### 下周计划：
调整 Python `matplotlib` 与前端 ECharts 的色彩映射模式，导出高对比度黑白灰兼用的论文图表版本；完成终稿排版，准备提交学校的官方查重系统。

## 第 13 周进展情况记录

### 本周完成/计划情况：
1. **系统降噪与学校查重（知网）**：提交毕设终稿至学校指定平台进行查重检验。针对个别在综述部分出现的常规重复句法进行学术化的缩减与同义词更迭，顺利将查重率压降至 5% 的优秀区间以下。
2. **毕设答辩材料精加工（PPT 制作）**：围绕“为什么做（痛点） -> 核心技术怎么做（CEI模型/物理隔离沙箱） -> 效果如何（A/B Test 覆盖率与机时对冲图）”为主轴，制作具备极强逻辑递进关系的高质量汇报演示 PPT。
3. **录制系统操作展示全流程回放视频**：由于答辩现场网络环境不可控（可能无法调用 DeepSeek API 或启动沉重的 WSL 编译环境），精心录制了一段长约 4 分钟的系统全链路操作演练实况录屏，作为答辩兜底。

### 存在的问题：
答辩汇报规定的时限通常为 8~10 分钟，目前准备的内容从前端页面讲解到底层 C++ 探针逻辑跨度极大，极易造成超时超时。

### 下周计划：
舍去部分非核心业务的增删改查前端介绍，将答辩着力点 80% 压实在“物理效能验证突破”和“大语言模型自闭环”这两个答辩评委最关注的技术深水区，进行多轮掐钟预演。
